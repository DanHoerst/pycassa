pycassa.connection – Non-pooled connections to Cassandra servers¶

Tools for connecting to a Cassandra cluster.

See also

Module pool to see how connections can be pooled.

To get a connection object which you can use directly with a ColumnFamily or with a ColumnFamilyMap, you can do the following:

>>> import pycassa
>>> connection = pycassa.connect('Keyspace', ['hostname:9160'])
pycassa.connection.connect(keyspace, servers=None, framed_transport=True, timeout=None, credentials=None, retry_time=60, recycle=None, round_robin=None, use_threadlocal=True)¶

Constructs a single Connection. Connects to a randomly chosen server on the list.

If the connection fails, it will attempt to connect to each server on the list in turn until one succeeds. If it is unable to find an active server, it will throw a NoServerAvailable exception.

Failing servers are kept on a separate list and eventually retried, no sooner than retry_time seconds after failure.

Parameters:
  • keyspace (str) – The keyspace to associate this connection with.
  • servers (str[]) – List of Cassandra servers with format: “hostname:port”. Default: ['localhost:9160']
  • framed_transport (bool) – If True, use a TFramedTransport instead of a TBufferedTransport. Cassandra 0.7.x uses framed transport, while Cassandra 0.6.x uses buffered.
  • timeout (float) – Timeout in seconds (e.g. 0.5). Default: None (it will stall forever)
  • retry_time (float) – Minimum time in seconds until a failed server is reinstated (e.g. 0.5). Default: 60.
  • credentials (dict) – Dictionary of user credentials. Example: {'username':'jsmith', 'password':'havebadpass'}
  • recycle (float) – Max time in seconds before an open connection is closed and replaced. Default: None (never recycle)
  • round_robin (bool) – DEPRECATED
Return type:

Connection

update_column_family(cfdef, block=True, sample_period=0.25)¶

Update a column family definition.

Warning

If you do not set values for everything in the CfDef, Cassandra will fill them in with bad values. You should get the existing CfDef using get_keyspace_description(), change what you want, and use that.

Example usage:

>>> conn = pycassa.connect('Keyspace1')
>>> cfdef = conn.get_keyspace_description()['MyColumnFamily']
>>> cfdef.memtable_throughput_in_mb = 256
>>> conn.update_column_family(cfdef)
Parameters:
  • cfdef (CfDef) – the column family definition
  • block – wait until the cluster has the same schema version to return
  • sample_period (int) – when blocking, how often to check for schema agreement among the cluster (in seconds)
Return type:

new schema version
